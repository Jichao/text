Examples

Hello, World!

Ok, here is the obligatory "Hello, world!" example: 

#include <boost/text/string.hpp>

#include <iostream>


int main ()
{
    boost::text::string_view const string_view("Hello, world!\n");
    boost::text::string const string(u8"всем привет!\n");

    std::cout << string_view;
    std::cout << string;

    return 0;
}

Pretty simple, we've just made a non-owning string_view, and an owning-and-allocating string. They each say "Hello, World!". The second one is what the Internet assures me is idiomatic Russian meaning the same thing.

Using Boost.Text Types in Interfaces

There are particular parameter passing conventions that some of the Boost.Text types are designed for. For instance, string_view, like std::string_view, is designed to bind to any char const *, string_view, or string argument, without making a copy. string_view is therefore a replacement for the multiple overloads it would take to handle all those types. If string_view was not implicitly convertible from char const * and string, this is the sort of code you would have to write: 

void print_string_without (boost::text::string_view str)
{ std::cout << str; }

void print_string_without (boost::text::string const & str)
{ print_string_without(boost::text::string_view(str)); }

void print_string_without (char const * str)
{ print_string_without(boost::text::string_view(str)); }

And here is what you can write with string_view's implicit conversions: 

void print_string_with (boost::text::string_view str)
{ std::cout << str; }

If you want to write a function parameter that handles any of Boost.Text's types, unencoded_rope_view is available: 

void print_any_string_type (boost::text::unencoded_rope_view str)
{ std::cout << str; }

And, sure enough, you can call it with any of the Boost.Text types, and even char const *: 

print_any_string_type("1\n");
print_any_string_type(boost::text::string_view("2\n"));
boost::text::string str_3("3\n");
print_any_string_type(str_3);
print_any_string_type(boost::text::repeat("4\n", 4));
boost::text::unencoded_rope rope_5("5\n");
print_any_string_type(rope_5);

Note

unencoded_rope_view is less efficient than string_view for binding to strings and string_views (though only a little). If you know you need that extra efficiency, and you know you don't need to support unencoded_ropes or repeated_string_views as arguments to your function, stick to string_view.

Ropes in Practice

A key use of ropes is in text editors. Every text editor you've ever used that handles large files gracefully has done something to break the file into chunks -- editing the beginning of a 16GB file that is stored in a contiguous sequence in memory is way too slow. The use of a rope is a common approach to remedy this.

In the example directory, there is a program rope_editor. It uses a rope to store the file you're editing. Even when typing at the beginning of a 1GB file, it's very snappy.

Note

These editors are pretty lame. They don't have cut/copy/paste, and can't even save your file! They use Emacs keybindings for what little they do.
