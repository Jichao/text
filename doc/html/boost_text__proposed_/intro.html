Intro

"It's all too much and not enough at the same time."

-- Jack Kerouac, referring to std::string

Motivation

std::string is an infamous design. It's fat -- why are there so many member functions? It's not fat enough -- what if I need to support non-ASCII characters? I could use std::basic_string<CharT> for some CharT that works for non-ASCII uses, but then I'm left using multiple types for strings — each of which is called string — without any encoding guarantees.

Apart from std::string in particular, text handling in C++ is problematic in general, due to the lack of Unicode support.

This library attempts to fill in some of these gaps in C++.

In part, this library is an attempt to re-think std::string as an interoperable family of types, each suited to a different aspect of string processing.

In part, this library is a Unicode implementation suitable for use by anyone — including those who know next to nothing about Unicode.

Features

- A family of types to replace std::string, for users that want "a better string" but do not care about encoding or Unicode. This includes string, string_view, repeated_string_view, unencoded_rope, and unencoded_rope_view.

- Iterators converting UTF-8 to and from UTF-16 and UTF-32.

- An implementation of the Unicode algorithms for dealing with encodings, normalization, etc. It is not necessary to use any of Boost.Text's string types to make use of the Unicode algorithms.

- A family of types for use in Unicode-aware text processing. This includes text, text_view, rope, and rope_view.

- Substrings of each of the string-like types above can be had with no copies or allocations using a Python-like slicing interface.

- Code simplicity -- none of the above string or text types is a template.
